# 模式选择指南

## 概述

本 skill 支持三种协作模式、三种执行深度、以及风险升级机制。根据任务特征和风险因子选择合适的组合。

## 第一步：评估基础复杂度

| 判定维度 | 轻量 | 标准 | 深度 |
|---------|------|------|------|
| 涉及文件数 | < 10 | 10-50 | > 50 |
| 涉及模块数 | 1 | 2-5 | > 5 |
| 跨服务/跨仓库 | 否 | 单仓库内 | 是 |
| 变更行数 | < 500 | 500-2000 | > 2000 |

**评估规则：** 满足任一维度的高级阈值即升级。取所有维度的最高级别。

## 第二步：扫描风险升级因子

**风险因子清单（任一出现即触发升级）：**

| 风险领域 | 关键词 |
|---------|--------|
| 安全边界 | authentication, authorization, tenant isolation, security boundary |
| 金融交易 | payment, billing, refund, charge |
| 数据完整性 | data deletion, bulk update, migration, schema change |
| 流程引擎 | workflow engine, approval flow, state machine |
| 契约稳定 | public API contract, production incident fix |

**升级规则：**
- 出现 **1 个**风险因子：轻量 → 标准
- 出现 **2 个或以上**风险因子：直接 → 深度
- **风险升级优先于文件数量判断**（高风险的小变更仍需深度评审）

**示例：**
```
变更范围：3 个文件（基础复杂度：轻量）
风险扫描：涉及 authentication + public API contract（2 个因子）
最终深度：深度（风险升级覆盖文件数量）
```

## 第三步：选择协作模式

### 快速决策矩阵

| 任务特征 | 推荐模式 | 理由 |
|---------|---------|------|
| 时间紧迫 / 有截止日期 | 集中调度 | 最快收敛 |
| 产品决策 | 集中调度 | 权责清晰 |
| 用户体验优化 | 领域主导（Designer） | UX 深度 |
| 技术架构深入评估 | 领域主导（Engineer） | 技术深度 |
| 产品策略/规划 | 领域主导（PM） | 产品深度 |
| 跨职能复杂权衡 | 对等协作 | 交叉碰撞 |
| 不确定 / 混合型 | 集中调度 | 最佳默认 |

### 决策树

```
是否涉及风险因子？
├── 是（>= 2 个）→ 深度评审 + 根据下文选择模式
├── 是（1 个）→ 标准评审 + 根据下文选择模式
└── 否 → 基础复杂度决定深度

是否有某个领域需要特别深入？
├── 是 → 领域主导模式
│   ├── UX → Designer 主导
│   ├── 技术 → Engineer 主导
│   └── 产品策略 → PM 主导
└── 否 → 时间是否紧迫？
    ├── 是 → 集中调度模式
    └── 否 → 是否涉及跨职能复杂权衡？
        ├── 是 → 对等协作模式
        └── 否 → 集中调度模式（默认）
```

## 第四步：选择自适应镜头（仅轻量模式）

轻量模式不使用固定的 PM/Designer/Engineer 角色，改为动态镜头组合。

### 镜头池

**核心评审者**（始终存在）：
负责整体事实收集、代码结构分析。

**风险镜头**（可选 1 个，根据变更内容选择）：

| 镜头 | 触发条件 | 聚焦 |
|------|---------|------|
| Security | 涉及认证、授权、输入处理 | 安全漏洞、注入、权限绕过 |
| Performance | 涉及循环、查询、缓存、并发 | 性能瓶颈、内存泄漏、N+1 |
| Reliability | 涉及错误处理、重试、超时 | 故障恢复、优雅降级 |
| Data Correctness | 涉及计算、转换、聚合 | 精度丢失、边界条件、数据一致 |
| UX Regression | 涉及 UI 组件、交互流程 | 可用性回退、无障碍退化 |
| API Compatibility | 涉及接口签名、序列化 | 破坏性变更、版本兼容 |

**上下文镜头**（可选 1 个，根据技术栈选择）：

| 镜头 | 触发条件 | 聚焦 |
|------|---------|------|
| Frontend | React/Vue/组件文件 | 组件状态、渲染性能、CSS |
| Backend | FastAPI/Django/路由文件 | 中间件、依赖注入、错误传播 |
| Database | SQL/ORM/迁移文件 | 索引、查询计划、数据完整 |
| DevOps | Docker/CI/配置文件 | 构建缓存、部署策略、环境变量 |
| Domain Expert | 特定业务逻辑 | 业务规则正确性 |

### 选择逻辑

Team Lead 分析变更内容后自动选择：

```
变更类型分析：
- 主要涉及的文件类型：xxx
- 主要涉及的功能域：xxx
- 主要涉及的风险域：xxx

选择结果：
- 核心评审者：始终
- 风险镜头 = [选定镜头]（理由：xxx）
- 上下文镜头 = [选定镜头]（理由：xxx）
```

### 示例

| 变更 | 风险镜头 | 上下文镜头 | 理由 |
|------|---------|-----------|------|
| 前端按钮文案修改 | 无 | Frontend | 纯 UI 变更，无功能风险 |
| 支付接口参数调整 | Security | Backend | 涉及支付，需安全审查 |
| 数据库迁移脚本 | Data Correctness | Database | Schema 变更，需数据完整检查 |
| API 响应格式变更 | API Compatibility | Backend | 接口签名变更，需兼容性检查 |

## 第五步：确定执行路径

### 轻量级执行

**标准/深度模式固定角色：**
```
Round 1: 3 agent 并行事实收集
  → [质量门禁 + 恢复路径]
Team Lead 直接裁决（跳过 Round 2）
  → [质量门禁]
行动计划
Agent 总数：3
```

**轻量模式自适应镜头：**
```
Round 1: 核心评审者 + 风险镜头 + 上下文镜头（2-3 agent）
  → [质量门禁 + 恢复路径]
Team Lead 直接裁决
  → [质量门禁]
行动计划
Agent 总数：2-3
```

### 标准级执行

**集中调度：** 3 + 1-2 + Team Lead = 4-5 agent
**领域主导：** 3 + 2-3 + Team Lead = 5-6 agent
**对等协作：** 3 + 4-5 + Team Lead = 7-8 agent

### 深度级执行

在标准级基础上，可选增加 Round 2.5（对争议领域补充调研）。

## 第六步：输出审批面板（强制）

Phase 0 完成后，向用户展示审批面板并**停止等待确认**：

```
=== 评审审批面板 ===
基础复杂度：轻量 / 标准 / 深度
风险升级因子：[列出检测到的因子] / 无
最终深度：轻量 / 标准 / 深度
协作模式：集中调度 / 领域主导 / 对等协作
选定 Agent：
  - [角色 1]：PM 标准版
  - [角色 2]：Designer 标准版
  - [角色 3]：Engineer 标准版
  - [评审组]：挑战者A + 挑战者B + 审计者
预估 Token：~XXXK
执行轮次：Round 1 / Round 1-2 / Round 1-3
跳过轮次：Round 2（理由：xxx）/ 无
自适应镜头：[仅轻量模式]
  - 风险镜头：xxx（理由：xxx）
  - 上下文镜头：xxx（理由：xxx）
====================
```

**豁免条件（用户明确说了以下之一可跳过审批）：**
- "continue automatically"
- "run full review"
- "no confirmation needed"

## 中途切换触发条件

| 触发条件 | 从 → 到 | 说明 |
|---------|---------|------|
| Round 2 出现 > 3 个关键质询 | 任何 → 对等协作 | 争议比预期大 |
| Round 1 某领域发现 > 70% 事实 | 任何 → 该领域主导 | 该领域是核心 |
| Round 2 争议密度 < 20% | 任何 → 集中调度式裁决 | 无需复杂辩论 |
| Round 2 争议密度 > 50% | 标准 → 深度 | 增加 Round 2.5 |
| 门禁反复失败（>= 3 次） | 任何 → 深度 | 需要更深入分析 |

切换时在画布"模式切换日志"中记录：时间、从/到、触发条件、理由。

## Token 消耗估算

| 执行路径 | Agent 数 | 预估 Token |
|---------|---------|-----------|
| 轻量级（固定角色） | 3 | ~50K-80K |
| 轻量级（自适应镜头） | 2-3 | ~40K-70K |
| 标准级（集中调度） | 4-5 | ~80K-150K |
| 标准级（对等协作） | 7-8 | ~150K-250K |
| 深度级 | 8-10 | ~250K-400K |
